# CoRDFL: Enhancing Statement-Level Fault Localization with LLMs through Dynamic Probing and Reflection

This repository contains the implementation and experimental artifacts for **CoRDFL**, a novel statement-level fault localization framework that integrates multi-round reflection with dynamic execution feedback to guide Large Language Models toward precise root-cause identification.

---

## 📌 Overview

Existing LLM-based fault localization approaches suffer from three critical limitations: coarse localization granularity, reliance on static code comprehension, and single-shot reasoning instability. CoRDFL addresses all three by constructing an iterative, closed-loop pipeline:

> **Fault Localization → Patch Generation → Patch Validation → Reflection**

Rather than treating generated patches as final repairs, CoRDFL uses them as **dynamic probes** — executing them against automated test suites to harvest concrete compilation and runtime feedback. This execution evidence is progressively fed back to the LLM across multiple rounds, enabling the model to self-evaluate and continuously refine its localization hypotheses.

Evaluated on **437 faulty versions across 17 real-world Java projects** from the Defects4J benchmark, CoRDFL achieves:
- **+92.0%** Top-1 improvement over LLMAO (static LLM baseline)
- **+128.4%** Top-1 improvement over SmartFL (probabilistic baseline) on Defects4J v1.2.0
- Consistent generalization on the newer Defects4J v3.0.1 benchmark

## 📂 Repository Structure

The repository is organized as follows:

 ├── ablation/                 # Results of ablation studies
 ├── code/                     # Core implementation of CoRDFL 
 ├── sourceofCodeContext/      # Source code context used for experiments
 ├── report/                   # Iterative experiment outputs and logs

### 📁 `ablation/`
Experimental results for four degraded variants used to quantify each component's contribution:
- `w/o Reflection Feedback` — severs inter-round dependency; Top-1 drops from 300 → 266
- `w/o Execution Outcome Feedback` — removes runtime signals; Top-1 drops to 277
- `w/o Multi-location Patch Exploration` — restricts probing to one candidate; Top-5 drops to 479
- `w/o Code-aware Reflection Context` — removes structural code semantics; Top-1 drops to 277

### 📁 `code/`
Core pipeline implementation, including:
- LLM-driven fault localization with line-aware code context construction
- Candidate patch generation using per-statement prompts
- Automated patch insertion, test suite execution, and safe file restoration
- Multi-round reflection synthesis and prompt augmentation logic

### 📁 `sourceofCodeContext/`
Pre-extracted function-level code contexts used as model inputs:
- Faulty method source code with line number annotations
- Structured inputs derived from Defects4J v1.2.0 and v3.0.1

### 📁 `report/`
Intermediate and final outputs from iterative experiments:
- Per-round Top-1/3/5 localization results across 7 reflection rounds
- Patch execution outcome logs (compilation failures, test deltas)
- Reflection summaries generated by the LLM at each iteration

## ⚙️ Experimental Setup

| Setting | Details |
|---|---|
| **Benchmark** | Defects4J v1.2.0 & v3.0.1 |
| **Projects** | 17 Java projects (e.g., Math, Closure, Lang, Jsoup) |
| **Faulty versions** | 437 versions / 711 faulty functions |
| **Task** | Statement-level fault localization |
| **LLM** | GPT-4o mini |
| **Max iterations** | 7 rounds (optimal trade-off at Round 3–4) |
| **Evaluation metrics** | Top-1, Top-3, Top-5 |

**Inputs per fault instance:**
- Faulty function source code (function-level context)
- Failing test cases and error logs

**Output:**
- Ranked list of suspicious statements with logical justifications

## 🔍 Key Design Decisions

**Dynamic Probing.** Candidate patches serve as active probes rather than repair attempts. Executing them against the test suite yields execution signals — compilation failures, increased/decreased/unchanged failing tests — each carrying distinct diagnostic weight. Patches that *reduce* failing tests achieve a **33.0% hit rate** for the true fault location, far exceeding other signal categories.

**Multi-Round Reflection.** The LLM synthesizes a structured reflection after each iteration, capturing causal relationships between patches and execution outcomes. This reflection is injected into the next round's prompt, enabling progressive hypothesis refinement rather than repeated single-shot guessing.

**Statement-Level Granularity.** Unlike most LLM-based FL methods that operate at file or method level, CoRDFL targets individual statements — providing the fine-grained guidance required for downstream automated program repair.

## 📊 Main Results

| Method | Top-1 (v1.2.0) | Top-3 (v1.2.0) | Top-5 (v1.2.0) |
|---|---|---|---|
| **CoRDFL** | **169** | 252 | 280 |
| SmartFL | 74 | **254** | **394** |
| LLMAO | 88 | 149 | 183 |

CoRDFL nearly doubles SmartFL's Top-1 accuracy and outperforms LLMAO by 92% at Top-1 — the metric most critical for developer trust and APR pipeline efficiency. SmartFL retains an advantage in broader Top-5 recall due to global trace coverage, suggesting complementary hybrid architectures as a promising future direction.

Iterative reflection improves performance substantially over single-round static prediction: average improvements of **+18.5 Top-1**, **+35.3 Top-3**, and **+37.8 Top-5** faults across rounds, with peak gains of up to 48.
